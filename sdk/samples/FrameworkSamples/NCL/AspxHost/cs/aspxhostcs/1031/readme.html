Übersicht

 

Die im Lieferumfang von Microsoft .NET 1.1 enthaltenen APIs können zum Hosten von ASPX-Anwendungen ohne IIS verwendet werden. Die im Lieferumfang der Beta 1-Version von .NET Framework 2.0 enthaltene HttpListener-Klasse wurde auf Grundlage von Http.Sys erstellt, sodass Benutzer einen eigenständigen HTTP-Server erstellen konnten. Diese Demo veranschaulicht, wie die Features von HttpListener kombiniert werden können, um einen HTTP-Server zu erstellen, der Aufrufe an die gehostete ASPX-Anwendung weiterleitet.

 

Erstellen des Beispiels

Die folgenden Schritte müssen durchgeführt werden, um dieses Beispiel korrekt zu erstellen und auszuführen.

Öffnen Sie die Projektdatei AspxHostCS.sln mithilfe von Visual Studio 2005 Beta.

Erstellen Sie dann das Projekt, indem Sie auf Erstellen -> Erstellen  Projekt klicken.

Ausführen des Beispiels

Befolgen Sie eine der nachfolgenden Anweisungen, um dieses Beispiel auszuführen.

  • Ausführen des Projekts von Visual Studio 2005 Beta aus
  • Öffnen Sie eine Eingabeaufforderung, und wechseln Sie zum Speicherort des Beispiels. Wechseln Sie an diesem Speicherort zum Verzeichnis bin\Debug. Führen Sie die Anwendung AspxHostCS.exe aus.

Wechseln Sie vor dem Ausführen des Beispiels zum Abschnitt Hinweise -> Verwendung des Beispiels.

Hinweise

Features des verwendetenHttpListener

 

  1. Authentifizierung
  2. Aktivieren von SSL
  3. Lesen von Clientzertifikaten über sichere Verbindungen

 

Klassenbeschreibung

 

AspxHostCS.cs è Die Hauptklasse, die einen Listener und eine ASPX-Anwendung erstellt und konfiguriert.

AspxVirtualRoot.cs è Klasse, die einen HttpListener zum Abhören von Präfixen und unterstützten Authentifizierungsschemas konfiguriert.

AspxNetEngine.cs è Klasse, die eine ASPX-Anwendung durch Zuweisen eines virtuellen Alias konfiguriert, der einem physikalischen Verzeichnis zugeordnet ist.

AspxPage.cs è Klasse, die die SimpleWorkerRequest-Klasse implementiert und eine vom Client angeforderte Seite darstellt.

AspxRequestInfo.cs è Datenhalterklasse, die zum Übergeben relevanter Daten von HttpListenerContext zur gehosteten Anwendung verwendet wird.

AspxException.cs è Benutzerdefinierte Ausnahmeklasse.

DemoPages èVerzeichnis, das Beispiel-ASPX-Seiten enthält.

 

Verwendung des Beispiels

 

Die Datei AspxHostCS.cs ist die Klasse, die die Hauptmethode enthält, die wiederum einen HttpListener startet und ein physikalisches Verzeichnis als gehostete ASPX-Anwendung konfiguriert. Standardmäßig versucht die Klasse, das Verzeichnis DemoPages (im selben Beispielverzeichnis) als gehostete Anwendung unter dem virtuellen Alias / zu konfigurieren. Wenn dies nicht geändert wird, wird durch Klicken auf den angegebenen Link diese Seite Infodatei verarbeitet.  Da der HttpListener in diesem Beispiel Anschluss 80 abhört, müssen Sie IIS möglicherweise stoppen, um dieses Beispiel ausführen zu können.

 

Ändern Sie den Code, um individuelle Anpassungen durchzuführen.

 

                //AspxVirtualRoot-Objekt mit einem HTTP-Anschluss und ggf. mit einem HTTPS-Anschluss erstellen.

                AspxVirtualRoot virtualRoot = new AspxVirtualRoot(80);

 

                //Physikalisches Verzeichnis als virtuellen Alias erstellen.

                //TODO: Physikalisches Verzeichnis mit dem zu konfigurierenden Verzeichnis ersetzen.

                virtualRoot.Configure("/", Path.GetFullPath(@"..\..\DemoPages"));

 

                //TODO: Wenn Authentifizierung hinzugefügt werden muss, diese hier einfügen

                //virtualRoot.AuthenticationSchemes = AuthenticationSchemes.Basic;

 

Festlegen des Authentifizierungsschemas

 

Legen Sie nach dem Konfigurieren eines AspxVirtualRoot-Objekts das erforderliche Authentifizierungsschema fest, indem Sie das Feld AuthenticationScheme auf dem AspxVirtualRoot-Objekt festlegen.

 

 

Aktivieren von SSL

 

Zum Aktivieren von SSL muss ein auf dem Computerspeicher installiertes Serverzertifikat auf dem Anschluss konfiguriert werden, auf dem SSL erforderlich ist. Weitere Informationen zum Konfigurieren eines Serverzertifikats auf einem Anschluss mithilfe des Dienstprogramms Httpcfg.exe finden Sie unter dem Link Httpcfg.

 

Hinweis: Winhttpcertcfg kann auch zum Konfigurieren eines Serverzertifikats auf einem Anschluss verwendet werden. Weitere Informationen finden Sie unter dem Link Winhttpcertcfg.

Häufig gestellte Fragen

 

Beim Starten der Anwendung erhalte ich die folgende Fehlermeldung:

 

 “System.IO.FileNotFoundException: Die Datei 'AspxHostCS, Version=1.0.1809.19805, Culture=neutral, PublicKeyToken=null' oder eine Abhängigkeit davon wurde nicht gefunden. Das System kann die angegebene Datei nicht finden. Dateiname: 'AspxHostCS, Version=1.0.1809.19805, Culture=neutral, PublicKeyToken=null'”

 

Lösung:

 

Die Datei AspxHostCs.exe ist im Verzeichnis bin des physischen Verzeichnisses, das konfiguriert wird, nicht vorhanden. Kopieren Sie die Datei AspxHostcs.exe in das Verzeichnis "bin".
